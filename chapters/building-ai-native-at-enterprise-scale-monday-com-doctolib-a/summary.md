# Summary: Building AI-native at enterprise scale: monday.com, Doctolib, and Delivery Hero

Chapter slug: `building-ai-native-at-enterprise-scale-monday-com-doctolib-a`

## Abstract

This panel discussion features engineering leaders from Monday.com, Doctolib, and Delivery Hero discussing their transition to AI-native enterprise operations using Claude. The companies, founded between 2011-2013, faced the challenge of integrating AI capabilities into existing monolithic codebases and established engineering organizations. Key implementations include Monday.com's Vibe application builder that converts prompts to working applications, Doctolib's skills marketplace enabling organization-wide Claude adoption, and Delivery Hero's HeroGen autonomous software delivery system achieving 85% pull request success rates. The discussion covers architectural decisions, model evaluation processes, and organizational adoption strategies. All three companies emphasize the importance of API-first architecture, standardized development patterns, and treating agents as first-class users in identity systems. The panel highlights how principal engineers are shifting from code review to hands-on development, orchestrating multiple agents for complex tasks. Success metrics focus on merge rates, quality indicators, and failure analysis to identify new use cases.

## Key claims

- Delivery Hero's HeroGen system achieves 85% success rate for autonomous pull request generation using a council of agents approach
- Doctolib has reached 100% adoption of Claude across engineering, product, and design teams through a skills marketplace approach
- Monday.com's Vibe can generate working applications from simple prompts in minutes by leveraging existing open platform APIs
- Model upgrades require treating new versions as completely different systems rather than compatible updates
- Smaller, standardized codebases significantly outperform monolithic systems for AI agent integration
- Principal engineers are shifting from code review to hands-on development using agent orchestration
- API-first architecture and agent-aware identity systems are critical for successful AI-native transformation
