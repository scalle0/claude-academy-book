# Summary: What's new in Claude Code

Chapter slug: `what-s-new-in-claude-code`

## Abstract

This presentation covers recent updates to Claude Code, focusing on developer experience improvements and autonomous capabilities. Developer experience enhancements include remote control functionality allowing session access across devices, a flickering-free full-screen terminal mode with virtualized scroll back, and a redesigned desktop application with project grouping and code review features. Autonomy features enable Claude to work with reduced human intervention through auto mode (which bypasses authorization requests for non-destructive actions), work trees for parallel development sessions, auto memory for session context retention, automated code reviews using multi-agent analysis, routines for scheduled workflows triggered by webhooks or timers, and agents view for managing multiple concurrent sessions. The platform now includes enterprise features such as improved Windows support, deployment wizards for Google Cloud Platform and AWS, and native binaries for governance pipelines.

## Key claims

- Remote control enables Claude Code session access from mobile devices and browsers with automatic session naming
- Full-screen mode eliminates terminal flickering through virtualized scroll back and enables clickable interface elements
- Auto mode reduces interruptions by automatically assessing whether actions are destructive or potentially malicious before requesting permissions
- Work trees provide native support for parallel Claude sessions on the same repository without file conflicts
- Auto memory automatically captures coding styles, architectural choices, and debugging insights in a memory.md file
- Code review implements multi-agent, multi-phase analysis to identify errors, bugs, and vulnerabilities in pull requests
- Routines enable scheduled Claude Code workflows triggered by webhooks, timers, or API calls without manual intervention
