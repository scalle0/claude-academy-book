# Summary: Code with Claude London 2026: Opening Keynote

Chapter slug: `code-with-claude-london-2026-opening-keynote`

## Abstract

This opening keynote from Code with Claude London 2026 presents Anthropic's roadmap for AI-assisted development across three product layers. The talk demonstrates how model capabilities are advancing exponentially, with examples including Mythos finding a 27-year-old OpenBSD vulnerability and agents now running for hours versus minutes previously. The presentation covers foundation model improvements with Opus 4.7 showing 3x better performance on production tasks, Claude platform enhancements including advisor strategy for cost reduction and managed agents with multi-agent orchestration, and Claude Code features like routines for asynchronous development and desktop app for multi-session management. Key architectural guidance emphasizes building for emerging capabilities rather than current limitations, reducing scaffolding as models become more intelligent, and treating model upgrades as business opportunities. The speakers highlight real customer deployments including Spotify's thousand-PR-per-month migration system and MercadoLibre's 23,000-engineer organization achieving autonomous coding workflows.

## Key claims

- API volume increased 60% year-over-year with 17x growth in code usage, with developers spending over 20 hours weekly in Claude Code
- Anthropic shipped 8 frontier models in the past 12 months, with Opus 4.7 performing 3x better than previous models on production engineering tasks
- Task horizon has expanded from minutes to hours, with future models expected to run continuously with proactive behavior
- Advisor strategy using smaller execution models with larger advisor models achieves frontier quality at 5x lower cost
- Claude Code adoption at Anthropic drove 200% increase in PRs per engineer despite organizational scaling
- Spotify's Claude-powered migration system merges over 1,000 PRs monthly into production with 90% time reduction
- As model intelligence increases scaffolding becomes limiting, requiring simplified architectures with generalized primitives
